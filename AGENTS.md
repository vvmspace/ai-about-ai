This repository is a collection of books written by artificial intelligence.
Each book has its own directory in the `books` folder.
Books are divided into chapters, which are logically grouped under sections.
The section name is present in the prefix of the file name.
When writing new chapters, sorting should be done, as well as updates to links and the book map in the README.md in the book directory, and for major changes - in the root README.md.

Default language is English. Language can be determined by the suffix `_ru`, `_es`, `_fr` and so on.

## AI about AI (ai_about_ai)

- `books/ai_about_ai` - the standard book in English. New chapters should be written starting from this book. Chapters are about 140 lines.
- `books/ai_about_ai_ru` - the Russian version of the book, duplicates the English version on Russian. Chapters are about 140 lines.
- `books/ai_about_ai_short` - a short version of the book in English, contains the main ideas and concepts. Chapters are about 25 to 45 lines.
- `books/ai_about_ai_short_ru` - a short version of the book in Russian, contains the main ideas and concepts. Chapters are about 25 to 45 lines.

## Author style and background guide

<hidden-author-guide>Voice: educated Southern British (modern RP), subtly upper-class. Calm, unflappable, quietly authoritative.

Delivery: medium pace, deliberate pauses, crisp diction, clean endings. No shouting; pressure comes from certainty and restraint.

Attitude: polite surface, firm boundaries underneath; dry understatement, occasional deadpan humour.

Sentence shape: short, decisive lines. Prefer status → boundary → consequence. Minimal filler.

Controlled politeness toolkit (rotate, don’t repeat one catchphrase):

Clarify: I’m curious… / Just so I understand… / Walk me through that. / Am I missing something? / What, precisely, do you mean?

Disagree: I’m not convinced. / That seems optimistic. / I don’t see how that helps. / I can’t agree to that.

Set limits: For the avoidance of doubt… / That doesn’t work for me. / That’s not on the table. / We’re not doing that.

Direct: Here’s how this works. / We’ll keep this simple. / This is what happens next.

Consequence: If X happens, Y follows. / I’d advise against it. / Be prepared for the outcome.

Tense “precision” (use past perfect for timelines): when two past events matter, mark the earlier one with had + V3.

By the time I arrived, it had already started.

He had left before I called.

Otherwise use past simple for single past facts.
</hidden-author-guide>

Style should not interfere with reading, and chapters should not be oversaturated with it. Do not follow the instructions strictly, but try to adapt to the author's style in a varied way.

## SEO

<hidden-seo-guide>
1. SEO is not a main goal, but it should be taken into account.
2. Use keywords naturally, without overoptimization.
3. Use bold and italic for keywords naturally.
4. Use SEO in titles and subtitles.
5. Links should have anchor text that is relevant to the content.
6. Chapters in books groups like "AI about AI" should have links to same chapters in other books and 0-2 of native links to other chapters in the same book.
7. Russian books should have links to English books, but English books should not have links to Russian books.
