# Стартовый промпт-сетап проекта (взгляд продукта)

Оператор спрашивает «контролируемо ли исполнение?», продукт-лид — «будет ли ценность для пользователя?»
Нужны оба ответа.

## Стартуем с user tension

Сначала одна фраза боли пользователя, потом уже промпты.

## Два фильтра качества

1. **Execution lens**: корректно, безопасно, стабильно.
2. **Value lens**: полезно, применимо, повышает доверие.

## Ранние сигналы

Для первых 50 сессий:

- использован ли ответ,
- потребовалось ли уточнение,
- вернулся ли пользователь в 24 часа.

## Главная мысль

Стартовый промпт-сетап — это продуктовая стратегия в технической форме.
